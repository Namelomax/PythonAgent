# DocumentAssistant

**DocumentAssistant** — это демонстрационный проект Retrieval-Augmented Generation (RAG),
который позволяет загружать документы, выполнять семантический поиск по их содержимому
и генерировать ответы на вопросы с использованием большой языковой модели (LLM).

Проект реализован как простой, читаемый и расширяемый пример RAG-пайплайна
без использования внешней базы данных.

---

## Возможности

- Загрузка документов в форматах **PDF**, **DOCX**, **TXT**
- Разбиение текста на фрагменты (chunking) с перекрытием
- Построение векторных представлений (embeddings) с помощью `sentence-transformers`
- Семантический поиск релевантных фрагментов по косиносовому сходству
- Генерация ответов с использованием LLM через **OpenRouter API**
- Хранение всех данных в памяти (in-memory)
- Сохранение результатов работы в `results.json`

---

## Архитектура проекта

Проект состоит из логически разделённых компонентов:

- **Document loaders** — загрузка и извлечение текста из файлов
- **Chunker** — разбиение текста на перекрывающиеся фрагменты
- **Embedding model** — преобразование текста в векторы
- **Retriever** — поиск наиболее релевантных фрагментов
- **LLM client** — генерация ответа на основе найденного контекста

## Требования

- Python **3.10** или выше

## Установка

1. Установка зависимостей из requirements.txt

Создать .env с ключом для LLM
OPENROUTER_API_KEY=your_api_key_here

2. Для проверки работоспособности можно запустить демонстрационный скрипт demo.py или использовать Jupyter-ноутбук demo.ipynb

## Возможные расширения
Проект легко расширяется за счёт замены in-memory хранения
на постоянное хранилище.

Например, можно подключить SurrealDB для хранения:

- текстовых фрагментов
- векторных embeddings
- истории запросов
- пользовательских сессий

SurrealDB поддерживает:

- хранение векторов
- работу с JSON-подобными структурами
- вычисляемые поля (например, timestamps)

Текущая архитектура проекта изначально спроектирована
с учётом таких расширений.
